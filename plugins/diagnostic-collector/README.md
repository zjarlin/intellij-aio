# Diagnostic Collector

自动收集项目中的编译错误和警告，分Tab页显示，支持一键复制生成AI修复提示词。

## 功能特性

- **自动收集诊断信息**：扫描项目源码中的编译错误和警告
- **分Tab页显示**：Errors 和 Warnings 分开展示，方便查看
- **文件级别聚合**：每行显示一个有问题的文件，包含问题数量和行号
- **快速导航**：点击文件名跳转到第一个问题位置
- **一键复制**：
  - 单文件复制：点击行左侧复制按钮
  - 批量复制：顶部工具栏提供全部复制、复制错误、复制警告按钮
- **AI修复提示词**：复制内容自动格式化为AI可理解的修复提示词

## 使用方法

1. 打开底部工具栏的 **Diagnostics** 面板
2. 点击 **刷新** 按钮收集当前项目的诊断信息
3. 在 Errors/Warnings Tab页中查看问题文件
4. 点击文件名跳转到问题位置
5. 点击复制按钮将诊断信息复制到剪贴板，粘贴给AI进行修复

## 复制格式示例

```
=== 文件: Example.kt ===
问题1:
  行号: 15
  类型: 错误
  内容: Unresolved reference: xxx
请帮我修复以上问题。
```

## 支持的文件类型

Java, Kotlin, Groovy, Scala, XML, JSON, YAML, Properties
