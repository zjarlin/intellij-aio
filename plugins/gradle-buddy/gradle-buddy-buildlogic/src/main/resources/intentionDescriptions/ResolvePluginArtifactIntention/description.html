Resolves the real implementation artifact of a Gradle plugin via the Plugin Marker Artifact mechanism.

When you declare id("org.jetbrains.kotlin.jvm") version "2.0.0" in a plugins { } block, this intention queries the Gradle Plugin Portal (and Maven Central as fallback) to find the actual implementation artifact (e.g. org.jetbrains.kotlin:kotlin-gradle-plugin:2.0.0).

The resolved artifact is written to libs.versions.toml under [libraries], so you can use implementation(libs.xxx) in your buildSrc or build-logic module to make the plugin available for precompiled script plugins.
