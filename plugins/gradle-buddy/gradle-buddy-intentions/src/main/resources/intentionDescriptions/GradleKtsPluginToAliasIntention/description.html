Converts hardcoded Gradle plugin declarations to catalog alias format.

Detects hardcoded plugin declarations with explicit version in plugins { ... } blocks and provides a quick action to convert them to use version catalog aliases. This helps modernize Gradle build scripts by centralizing plugin management in version catalogs.

This intention only applies to plugin declarations that include an explicit version statement. Precompiled script plugins (which cannot declare versions) are not supported.

Before: 


plugins {
    id("site.addzero.gradle.plugin.repo-buddy") version "+"
    id("site.addzero.gradle.plugin.addzero-git-dependency") version "2026.02.02"
}

After applying the intention: 


plugins {
    alias(libs.plugins.site-addzero-gradle-plugin-repo-buddy)
    alias(libs.plugins.site-addzero-gradle-plugin-addzero-git-dependency)
}

The intention automatically updates libs.versions.toml: 


[versions]
site-addzero-gradle-plugin-repo-buddy = "+"
site-addzero-gradle-plugin-addzero-git-dependency = "2026.02.02"

[plugins]
site-addzero-gradle-plugin-repo-buddy = { id = "site.addzero.gradle.plugin.repo-buddy", version.ref = "site-addzero-gradle-plugin-repo-buddy" }
site-addzero-gradle-plugin-addzero-git-dependency = { id = "site.addzero.gradle.plugin.addzero-git-dependency", version.ref = "site-addzero-gradle-plugin-addzero-git-dependency" }

Supports plugin declaration formats: id("plugin.id") version "version" and kotlin("module") version "version".

Works with Gradle version catalogs and helps migrate from hardcoded plugin declarations to the modern alias-based approach.
