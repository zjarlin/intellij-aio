Converts hardcoded dependency strings to TOML format and replaces them with version catalog references.

Detects hardcoded dependency declarations in build.gradle.kts files and provides a quick action to convert them to version catalog format. This helps modernize Gradle build scripts by centralizing dependency management in libs.versions.toml.

Before: 


dependencies {
    implementation("site.addzero:tool-curl:0.0.672")
    implementation("com.squareup.okhttp3:okhttp:4.12.0")
}

After: 


dependencies {
    implementation(libs.tool-curl)
    implementation(libs.okhttp)
}

The intention also generates the corresponding TOML library declarations as comments that can be copied to the libs.versions.toml file: 


// TOML format:
// [libraries.tool-curl]
// group = "site.addzero"
// name = "tool-curl"
// version = "0.0.672"

This intention supports all common dependency configurations: implementation, api, compileOnly, runtimeOnly, testImplementation, and more.

Works with Gradle version catalogs and helps migrate from traditional dependency declarations to the modern TOML-based approach.
