Converts Gradle plugin declarations to TOML format for version catalogs.

Detects plugin declarations in plugins { ... } blocks and provides a quick action to convert them to TOML format. This helps modernize Gradle build scripts by centralizing plugin management in version catalogs.

Before: 


plugins {
    id("site.addzero.gradle.plugin.kotlin-convention") version "2025.12.20"
    kotlin("jvm") version "1.9.10"
    alias("my-alias") version "1.0.0" applied false
}

After applying the intention, TOML format comments are generated: 


plugins {
    id("site.addzero.gradle.plugin.kotlin-convention") version "2025.12.20"
    // TOML format:
    // [plugins.site-addzero-gradle-plugin-kotlin-convention]
    // id = "site.addzero.gradle.plugin.kotlin-convention"
    // version = "2025.12.20"

    kotlin("jvm") version "1.9.10"
    // TOML format:
    // [plugins.kotlin-jvm]
    // id = "org.jetbrains.kotlin.jvm"
    // version = "1.9.10"
}

The generated TOML format can be copied to the libs.versions.toml file: 


[plugins]
site-addzero-gradle-plugin-kotlin-convention = { id = "site.addzero.gradle.plugin.kotlin-convention", version = "2025.12.20" }
kotlin-jvm = { id = "org.jetbrains.kotlin.jvm", version = "1.9.10" }

Supports various plugin declaration formats: id("plugin.id") version "version", kotlin("jvm") version "version", and alias("name") version "version".

Works with Gradle version catalogs and helps migrate from traditional plugin declarations to the modern TOML-based approach.
